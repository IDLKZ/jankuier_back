Создадим seeders для ProductOrderItemStatusEntity

1. Заказ создан, ожидает оплаты (prev_id = null, next_id = 2)
2. Заказ оплачен, ожидает подтверждения (prev_id = 1, next_id = 3)
3. Заказ доставляется (prev_id = 2, next_id = 4)
4. Заказ ожидает подтверждения доставки (prev_id = 3, next_id = 5)
5. Заказ успешно получен (prev_id = 4, next_id = null)
6. Заказ отменен (prev_id = 1, next_id = null)
7. Заказ отменен, ожидает возврата средства (prev_id = 6, next_id = 8) но может поддерживать 2,3,4 -> previous_allowed_values
8. Заказ отменен, средства возвращены (prev_id = 7, next_id = null)
